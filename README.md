> 📌 **Writing Rule (중요)**
>
> 이 문서는 “무엇을 배웠는가”를 기록하지 않는다.  
> 대신 **왜 이 문제를 다루었고, 어떤 판단을 거쳐 현재 결론에 도달했는지**를 기록한다.
>
> - 정답을 찾는 과정 ❌  
> - 사고와 선택의 흔적 ⭕
> - 실패와 포기 이유 ⭕
>
> 이 문서는 **미래의 나와, 이 레포를 읽는 타인에게  
> “왜 이런 설계를 했는지”를 설명하기 위한 기록**이다.
>
> 완벽한 결론이 없어도 된다.
> **지금 시점의 판단**이면 충분하다.

## 작성 템플릿

### 왜 이걸 다루는가
- 이 문제가 왜 중요하다고 판단했는지
- 해결하지 않으면 어떤 한계가 생기는지

### 처음 생각
- 문제를 처음 마주했을 때의 가설
- 직관적으로 맞다고 믿었던 방향

### 시도 / 비교
- 고려했던 선택지들
- 각 선택지의 장단점
- 왜 특정 선택을 했는지

### 깨진 지점
- 예상과 달랐던 부분
- 실패한 이유
- 이 선택이 위험하거나 한계가 있다고 느낀 근거

### 현재 판단
- 지금 시점에서의 결론
- 왜 이 판단이 가장 합리적이라고 보는지
- 임시 결론이라면 그 이유

### 남은 질문
- 아직 풀리지 않은 의문
- 다음 단계에서 검증해야 할 것
- 언제 다시 이 문제를 다룰지
